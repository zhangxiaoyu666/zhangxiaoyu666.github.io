java之加密与安全

发表于 2019-10-30 分类于 Java

数据安全

  • 防窃听
  • 防篡改
  • 防伪造

古代加密方法

  • 移位密码 : HELLO=>IFMMP（所有的字母都后移一位）
  • 替代密码 : HELLO=>P12,5,3（某一本书的第几页第几行第几个单词）

现代计算机加密

  • 建立在严格的数学理论基础上
  • 密码学逐渐发展成一门科学

编码算法

URL编码

什么是URL

URL编码是编码算法，不是加密算法

URL编码是浏览器发送数据给服务器时使用的编码

URL编码规则

URL编码的目的是把任意文本数据编码为%前缀表示的文本

  • A-Z,a-z，0-9以及-_.*保持不变
  • 其他字符以%XX表示(十六进制)
    • < : %3C
    • 中：%E4%B8%AD(UTF-8:0xe4b8ad)

Base64编码算法

什么是Base64

是一种把二进制数据用文本表示的编码算法

Base64编码

Base64编码目的及应用

目的：

  • 一种用文本(A-Z,a-z,0-9,+/=)表示二进制内容的方式
  • 适用于文本协议
  • 效率下降，因为二进制经过Base64编码，长度会增加1/3

应用：

  • 电子邮件协议

如果长度不是3的整数倍：

  • 末尾补0x00或0x00 0x00
  • 编码后加=表示补充了一个字节
  • 编码后加==表示补充了两个字节

这样解码时就可以删除补充的字节

java中使用URL的Base64编码，会将+变为-，/变为_,这样在传递URL参数时，不会发生冲突

摘要算法MD5

什么是摘要算法

摘要算法(哈希算法、Hash、Digest、数字指纹)：

  • 计算任意长度数据的摘要（固定长度）
  • 相同的输入数据始终得到相同的输出
  • 不同的输入数据尽量得到不同的输出

目的

  • 验证原始数据是否被篡改

碰撞

两个不同的输入得到了相同的输出

碰撞是不能避免的，因为输入的无限的，输出的位数是有限的，哈希算法就是将无限的输入映射到有限的数据上，所以一定会发生碰撞。

安全性

  • 碰撞率低
  • 不能猜测输出
  • 输入的任意一个bit的变化会造成输出完全不同
  • 很难从输出反推输入(只能依靠暴力穷举)

常用的摘要算法

Java中使用MD5算法

输入的数据可以分片

MD5用途

  • 验证文件完整性
  • 存储用户口令

  • 使用MD5存储口令时要小心彩虹表攻击

抵御彩虹表攻击：对每个口令额外添加随机数salt

  • Java中的hashCode()方法

使用

引入随机数salt

SHA-1分类

加密和解密使用同一个密钥，例如WinRAR

对称加密算法
